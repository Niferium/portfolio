Clementine AI

Clementine is a locally-deployed AI assistant designed to provide secure, private, and memory-enhanced support for both personal and enterprise use. Built as the successor to the prototype Erin, Clementine adopts a modular, Android-inspired architecture (e.g., ViewModel, Repository, DI) to keep components organized, testable, and maintainable.
It runs on top of DeepSeek R1 integrated via Ollama, with vector-based memory (RAG) for context-aware responses. Clementine is planned to feature a FastAPI backend, a lightweight Gradio web UI, and optional Android and mobile-friendly front-ends.
The project emphasizes local operation, data privacy, and extensibility, making it suitable for professional workflows—such as development assistance, secure knowledge retrieval, and task automation—without relying on external cloud LLM services.

Digital Bingo Game System

A full-stack web application built to modernize physical bingo events — bringing real-time card tracking, live number draws, and instant winner validation all into one seamless platform. Key Features: Card Registration — Players register their physical bingo cards online before the event starts, entering their Card ID, full name, mobile number, and all 24 card numbers. A countdown timer enforces the registration deadline. Host Draw Panel — The game host draws numbers live through a dedicated panel, which visually tracks all called numbers on a full 1–75 bingo board with color-coded column highlights (B-I-N-G-O). Registered card count and draw history are displayed in real time. Moderator Verification Panel — When a player claims bingo, the moderator enters the Card ID to instantly verify whether the claim is valid. The system cross-checks drawn numbers against the registered card and returns a clear VALID or INVALID result — with the card grid highlighted to show matched numbers. Live Draw Viewer — A public-facing screen displays the current number, all previously called numbers with timestamps, and an auto-refreshing bingo board — ideal for projecting during live events. Stack: React · Node.js · Persistent storage for card registry and draw state · Mobile-friendly registration flow
